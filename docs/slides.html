The Problem

  • AI agents make the same mistakes across sessions
  • Corrections you give are lost when the session ends
  • Teammates repeat the same trial-and-error with their agents
  • No way to measure what's actually working (or failing)
🚀

Quickstart

# 1. Install
git clone https://github.com/RetroCode-Org/RetroCode
cd RetroCode && pip install -e .

# 2. Set your API key
export COMMONSTACK_API_KEY=your_key_here

# 3. Go to your project and run
cd ~/my-project
retro --offline --dir .

That's it. Your CLAUDE.md now has learned rules.

Running Modes

ModeCommandBehavior
One-passretro --offlineProcess all new sessions, update playbook, exit
Interactiveretro --offline --verboseReview each change before applying
Daemonretro --upBackground polling — auto-updates continuously
Debugretro --foregroundDaemon in foreground — see all output live

Stop the daemon with retro --down

Skills Sharing

Dev A
export-skills
→
.tar.gz
portable bundle
→
Dev B
import-skills
# Developer A: generate and bundle
retro --export-skills -o my-skills.tar.gz

# Developer B: import from the bundle
retro --import-skills -i my-skills.tar.gz

Skills encode non-obvious repo knowledge — extension points, debugging workflows, architectural patterns — things an agent can't discover alone.

Import Merge Strategies

StrategyFlagWhat Happens
Local-first--strategy local-firstKeep your version, skip shared (default)
Retro-first--strategy retro-firstOverwrite local with shared version
Merge--strategy mergeKeep local, append new sections from shared
# Merge: best of both worlds
retro --import-skills -i teammate.tar.gz --strategy merge

Hypothesis Generation

Statistically test which agent behaviors predict user rejection.

retro --hypogen --dir .     # find patterns
retro --submit --dir .      # share with community
retro --pull --dir .        # verify community patterns

Example finding:

[sig] edit_without_read   p=0.003  OR=2.45
  "Agent edits a file it never Read" → 2.4x more likely to be rejected
