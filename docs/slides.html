The Problem → The Fix

Without RetroCode
session 1
[user] Fix the auth bug [agent] I'll edit auth.py directly... [user] No! Read it first. You broke the existing logic. ── 3 days later, different session ── [user] Fix the payment bug [agent] I'll edit payment.py... [user] No! Same mistake again! You didn't read the file!

Same corrections. Every session. Lost forever.

With RetroCode
CLAUDE.md (auto-generated)
## CODING_PATTERNS [coding-00004] Always Read a file before editing it to avoid overwriting existing logic. ## COMMON_MISTAKES [mistake-00009] Validate provider config on startup before daemonizing.
session 2 (auto-corrected)
[user] Fix the payment bug [agent] I'll read payment.py first... Found the issue on line 42. Fixed. [user] Perfect. ✔

Learns once. Applies forever. Across all sessions.

Four Capabilities

📖

Playbook Generation

Extracts rules from misunderstandings in your traces. Writes them into CLAUDE.md, .cursor/rules/, AGENTS.md.

🤝

Skills Sharing

Auto-generates repo-specific agent skills. Bundle & share with teammates via -o / -i flags.

🔬

Hypothesis Testing

Chi-squared tests on agent behavior patterns. Finds what predicts rejection with p-values and odds ratios.

🎵

AnalyzeMe Wrapped

Spotify-style coding stats: persona, toolkit, schedule, patience score, fun facts. Zero LLM calls.

Get Running in 60 Seconds

# 1. Install
git clone https://github.com/RetroCode-Org/RetroCode
cd RetroCode && pip install -e .

# 2. Set your API key (pick one)
export COMMONSTACK_API_KEY=your_key   # free credits for members
# or: OPENAI_API_KEY, ANTHROPIC_API_KEY, GEMINI_API_KEY

# 3. Run on your project
cd ~/my-project
retro --offline --dir .               # one-pass update
retro --offline --verbose --dir .     # interactive (approve each)
One-pass retro --offline
Daemon retro --up
Debug retro --foreground

Smart Merge In Action

Content-aware section merge

Same section heading? Unique items are merged, duplicates skipped:

local ## Steps
## Steps - Create the reader file ← kept - Register in __init__.py ← kept
+ merge
shared ## Steps
## Steps - Create the reader file ← skipped (dupe) - Add tool name mappings ← NEW, appended - Add config support ← NEW, appended
=
result
## Steps - Create the reader file - Register in __init__.py - Add tool name mappings ✔ - Add config support ✔
